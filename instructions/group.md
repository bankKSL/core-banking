## 1. Overview

The **Groups** feature implements back-office management of _self-help groups_ (a.k.a. lending
groups) in Apache Fineract. A group is an organizational unit that sits between an **Office** and
its member **Clients**, and is used primarily for **Joint Liability Group (JLG)** lending and group
(collection-sheet) repayments.

The UI provides two screens:

| Screen                                 | Component             | Route                                |
| -------------------------------------- | --------------------- | ------------------------------------ |
| Groups list (search / sort / paginate) | `GroupsListComponent` | `/groups`                            |
| Create / Edit / Activate a group       | `GroupFormComponent`  | `/groups/create`, `/groups/edit/:id` |

Both are **standalone Angular components**, lazy-loaded through `app.routes.ts` and guarded by
`authGuard` (the routes are children of the authenticated `MainLayoutComponent`).

### Files

| File                           | Responsibility                                                               |
| ------------------------------ | ---------------------------------------------------------------------------- |
| `groups-list.component.ts`     | Read-only data table of groups with server-side search, sort and pagination. |
| `group-form.component.ts`      | Create, edit and activate a group (template-driven `ngForm`).                |
| `group-form.component.spec.ts` | Unit tests for the form component.                                           |

## 3. Business Logic

### 3.1 Groups List (`GroupsListComponent`)

The list is a thin reactive wrapper around the shared `DataTableComponent`. It maintains three
pieces of local state and merges them into a single reactive stream that triggers one API call per
change:

- **Search** — a free-text `name` filter. Resets to page 0.
- **Sort** — `active` column + `direction` (`asc`/`desc`). Resets to page 0.
- **Pagination** — `pageIndex` + `pageSize` (default page size **10**).

Implementation notes:

- `merge(searchSubject, sortSubject, pageSubject)` + `startWith({})` fires an initial load.
- `switchMap` cancels any in-flight request when a new search/sort/page event arrives, so only the
  latest query is honoured.
- Request parameters are derived as:
  - `offset = pageIndex * pageSize`
  - `limit  = pageSize`
  - `orderBy  = sort.active || undefined`
  - `sortOrder = sort.direction.toUpperCase() || undefined` (`ASC` / `DESC`)
  - `name = currentFilter || undefined`
  - `paged = true` (always — the UI relies on `totalFilteredRecords` for the paginator)
- `catchError(() => of(null))` swallows HTTP errors and renders an empty table (the global
  error interceptor surfaces the message).
- On response, `totalRecords = response.totalFilteredRecords || 0` and the table data is
  `Array.from(response.pageItems)` (the model is a `Set`, converted to an array).

Columns displayed: `accountNo`, `name`, `status` (rendered via `StatusBadgeComponent`),
`officeName`, and an `actions` column containing only an **Edit** button.

> Note: the column key `accountNo` is declared in the table but is **not** part of the
> `GetGroupsPageItems` model, so it renders empty. The available fields are
> `id, name, officeId, officeName, active, hierarchy, status`.

### 3.2 Group Create / Edit / Activate (`GroupFormComponent`)

A single template-driven (`FormsModule` / `ngForm`) form handles three distinct operations, driven
by route state and the entity's persisted `active` flag.

#### Mode detection

- `ngOnInit` reads `route.paramMap`. If an `id` param is present → **edit mode**
  (`isEditMode = true`, `groupId = +id`) and the group is loaded via `getGroupsGroupId`.
- No `id` → **create mode**.
- `originalActive` captures the _persisted_ `active` value from the server (read from the raw
  response because the generated `GetGroupsGroupIdResponse` model does not type the `active` field).
  This is used to decide whether the **Activate** action is offered.

#### Field rules

| Field            | Create mode                               | Edit mode                                      |
| ---------------- | ----------------------------------------- | ---------------------------------------------- |
| `name`           | editable, required                        | editable, required (only field sent on update) |
| `officeId`       | editable, required (dropdown of offices)  | **disabled** (a group's office cannot change)  |
| `activationDate` | shown; required only when `active = true` | **hidden** (handled by the Activate action)    |
| `active`         | editable checkbox (defaults to `true`)    | **disabled**                                   |

#### Create flow (`onSubmit`, `isEditMode = false`)

1. Formats `activationDate` (`Date` → `yyyy-MM-dd`) manually.
2. Builds the payload by spreading the `group` model and **adding three mandatory fields that the
   generated `PostGroupsRequest` model omits** (`activationDate`, `dateFormat`, `locale`). The model
   only types `{ active?, name?, officeId? }`, so the payload is cast via `Record<string, unknown>`.
3. Calls `POST /v1/groups` (`postGroups`).
4. On success navigates back to `/groups`; on error stops the spinner (`isSaving = false`).

Fineract server rule enforced by this payload: _if `active = true`, `activationDate` is mandatory_.

#### Edit flow (`onSubmit`, `isEditMode = true`)

1. Builds a minimal payload containing **only `name`** (`PutGroupsGroupIdRequest = { name? }`).
2. Calls `PUT /v1/groups/{groupId}` (`putGroupsGroupId`).
3. On success navigates back to `/groups`.

#### Activate flow (`onActivate`) — edit mode only, group currently pending

- The **Activate Group** button is rendered only when `isEditMode && !originalActive` (i.e. the
  group exists and is still in a _pending_ state).
- Uses the `activationDate` field (defaults to today) and the command-style endpoint:
  `POST /v1/groups/{groupId}?command=activate` with body
  `{ activationDate, dateFormat: 'yyyy-MM-dd', locale: 'en' }` via `postGroupsGroupId`.
- On success it updates local state in place (`originalActive = true`, `group.active = true`) so the
  Activate button disappears **without navigation**.

#### Date formatting

`DATE_FORMAT = 'yyyy-MM-dd'`; both create and activate format the date with manual
`getFullYear()/getMonth()+1/getDate()` zero-padding (not locale-dependent `toISOString`, to avoid
timezone shifts).

## 4. API Calls

All endpoints are exposed through the generated **`GroupsService`** (and **`OfficesService`** for
the office dropdown). The resolved base URL is `BASE_PATH`, provided in `app.config.ts` from
`ConfigService.apiUrl` (the `/v1` suffix is trimmed), so every request is prefixed with
`{fineractApiUrl}` — e.g. `/fineract-provider/api` in dev (proxied to
`https://127.0.0.1:8443` per `proxy.conf.json`). All calls require HTTP Basic auth
(`Authorization: Basic …`) and the `fineract-platform-tenantid` header.

### 4.1 Endpoints used by the feature

| #   | Method & Endpoint                            | Service method                                 | Used in         | Purpose                               |
| --- | -------------------------------------------- | ---------------------------------------------- | --------------- | ------------------------------------- |
| 1   | `GET /v1/groups`                             | `getGroups(...)`                               | List            | Paged/sorted/filtered list of groups. |
| 2   | `GET /v1/groups/{groupId}`                   | `getGroupsGroupId(groupId)`                    | Form (edit)     | Load a single group for editing.      |
| 3   | `POST /v1/groups`                            | `postGroups(body)`                             | Form (create)   | Create a group.                       |
| 4   | `PUT /v1/groups/{groupId}`                   | `putGroupsGroupId(groupId, body)`              | Form (edit)     | Update a group's name.                |
| 5   | `POST /v1/groups/{groupId}?command=activate` | `postGroupsGroupId(groupId, body, 'activate')` | Form (activate) | Activate a pending group.             |
| 6   | `GET /v1/offices?includeAllOffices=true`     | `getOffices(true)`                             | Form            | Populate the office dropdown.         |

### 4.2 Request/response details

#### 1) List groups — `GET /v1/groups`

Call site (`groups-list.component.ts`):

```ts
this.groupsService.getGroups(
  undefined, // officeId
  undefined, // staffId
  undefined, // externalId
  nameFilter, // name            <- search text
  undefined, // underHierarchy
  true, // paged
  offset, // pageIndex * pageSize
  limit, // pageSize
  orderBy, // sort column
  sortOrder, // 'ASC' | 'DESC'
);
```

Full signature: `getGroups(officeId?, staffId?, externalId?, name?, underHierarchy?, paged?,
offset?, limit?, orderBy?, sortOrder?, orphansOnly?)`.

Response `GetGroupsResponse`:

```ts
{
  totalFilteredRecords?: number;
  pageItems?: Set<GetGroupsPageItems>;
}
```

`GetGroupsPageItems` = `{ id?, name?, officeId?, officeName?, active?, hierarchy?, status? }` where
`status: GetGroupsStatus = { id?, code?, description? }` (the badge renders the status `code`/
`description` provided by the backend).

#### 2) Retrieve a group — `GET /v1/groups/{groupId}`

```ts
this.groupsService.getGroupsGroupId(this.groupId); // optional: staffInSelectedOfficeOnly?, roleId?
```

Returns `GetGroupsGroupIdResponse = { id?, name?, officeId?, officeName?, externalId?, hierarchy?,
timeline? }`. The component also reads the untyped `active` flag from the raw response object.

#### 3) Create a group — `POST /v1/groups`

Server contract: **mandatory** `name`, `officeId`, `active`, and `activationDate` (when
`active = true`); **optional** `externalId`, `staffId`, `clientMembers`.

Payload actually sent (superset of the generated `PostGroupsRequest` model):

```json
{
  "name": "Test Group",
  "officeId": 1,
  "active": true,
  "activationDate": "2026-05-09",
  "dateFormat": "yyyy-MM-dd",
  "locale": "en"
}
```

Returns `PostGroupsResponse` (contains the new `groupId` / `resourceId`).

#### 4) Update a group — `PUT /v1/groups/{groupId}`

Payload (`PutGroupsGroupIdRequest`) — only the name is editable:

```json
{ "name": "Renamed Group" }
```

Returns `PutGroupsGroupIdResponse`.

#### 5) Activate a group — `POST /v1/groups/{groupId}?command=activate`

```ts
this.groupsService.postGroupsGroupId(
  groupId,
  { activationDate: "2026-05-09", dateFormat: "yyyy-MM-dd", locale: "en" },
  "activate",
);
```

Returns `PostGroupsGroupIdResponse`. Errors if the group is already active.

#### 6) List offices — `GET /v1/offices?includeAllOffices=true`

```ts
this.officesService.getOffices(true); // includeAllOffices
```

Returns `GetOfficesResponse[]` used to fill the Office `<mat-select>`.

### 4.3 Other `GroupsService` endpoints (available but NOT used by this UI)

The generated service also exposes the following, which the current screens do not call:

- `GET /v1/groups/{groupId}/glimaccounts` — GLIM accounts.

- `GroupsLevelService` (`/v1/groups/levels`) — group-level reference data (separate service).

## 7. Known Gaps / Observations

- **No delete UI** — `deleteGroupsGroupId` exists in the API but is not surfaced.
- **No detail/view screen** — groups are only edited inline; there is no read-only view, no client
  membership management, no collection-sheet UI, and no staff assignment UI (all available via the
  command endpoint but unused).
- **`accountNo` column** is declared in the list but absent from the response model, so it always
  renders blank.
- **Untyped `active` field** on the detail response forces a `Record<string, unknown>` cast in the
  component, and the create/activate payloads require fields (`activationDate`, `dateFormat`,
  `locale`) that the generated request models do not declare — a sign the OpenAPI spec under-documents
  the activation contract.
- **Error handling** relies on the global error interceptor; the list degrades to an empty table and
  the form simply re-enables the save button on failure.
